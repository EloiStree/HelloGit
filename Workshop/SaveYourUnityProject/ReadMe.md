**Sauver votre projet avec Git et Unity3D**

* [ ] Afficher les dossiers cachés
* [ ] Afficher les extensions de fichiers
* [ ] Créer un compte GitHub
* [ ] Créer un répertoire privé ou public
  * [ ] Ajouter un README
  * [ ] Attention à l’alphanumérique
* [ ] Télécharger l’application GitHub
* [ ] Cloner le projet avec l’application GitHub sur le PC
* [ ] Déplacer votre projet dans le répertoire
* [ ] Ajouter un `.gitignore` à la racine
  * [ ] [https://www.toptal.com/developers/gitignore/api/unity](https://www.toptal.com/developers/gitignore/api/unity)
  * [ ] Vérifier que le `.gitignore` fonctionne
* [ ] Vérifier qu’il n’y a pas de fichier de plus de 20 Mo
  * [ ] Télécharger [WinDirStat](https://windirstat.net)
  * [ ] Vérifier la taille des fichiers
  * [ ] ⚠️ Commiter un gros fichier est une calamité à retirer du projet
    * [ ] Mettre en ligne un gros fichier est encore pire à corriger
* [ ] Sauver les fichiers
* [ ] Récupérer les fichiers en ligne
* [ ] Pousser les fichiers sauvegardés en ligne
* [ ] Ta-da ;)

**Tester de meilleurs outils :**

* [ ] Télécharger Fork
* [ ] Télécharger SourceTree

**Pour pratiquer :**

* [ ] Modifier un fichier dans votre projet
* [ ] Aller dans Fork
* [ ] Ajouter la nouvelle modification à la prochaine sauvegarde
  * [ ] Ajouter un commentaire avant de sauvegarder
* [ ] Aller modifier le README sur GitHub de votre répertoire
* [ ] Récupérer sur votre ordinateur
* [ ] Pousser votre nouvelle sauvegarde en ligne
